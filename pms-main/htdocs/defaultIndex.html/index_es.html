Oracle HTTP Server 12c  




Oracle HTTP Server 12c se basa la tecnología probada de servidor HTTP Apache de código abierto y proporciona el marco para albergar páginas web dinámicas y estáticas, además de aplicaciones de Oracle Fusion Middleware para usuarios.






Funciones


Entrega de Contenido/Proxy Inverso

Despliegue en Nube/Soporte de Servidor Virtual
Miles de sitios/dominios de aplicación a partir de una única instancia del servidor web. Cada servidor virtual puede tener sus propios archivos de configuración, direcciones IP, puerto, raíz del documento, preferencias, archivos log, etc.

Protección frente a amenazas comunes
El módulo ModSecurity incorporado permite configurar reglas para la introspección y proteger las aplicaciones de ataques comunes, entre otros, inyección de SQL/comandos, Cross Site Scripting y otras vulnerabilidades.

Soporte para FastCGI
Eficaz método para servir páginas web de contenido dinámico en OHS utilizando lenguajes de script como PHP o Python, sin que suponga una pérdida significativa de rendimiento.

Proxy Inverso Integrado
Los módulos de proxy integrados proporcionan soporte de proxy genérico, además de optimizado para WebLogic Server, permitiendo así que OHS actúe como punto final HTTP para los servidores de origen HTTP, entre otros, WebLogic Server.

Administración/Supervisión

Administración de Servidor
Aprovecha las interfaces de administración de WebLogic 12c para proporcionar un modelo de administración distribuida simple y consistente para administrar Oracle HTTP Server, Oracle WebLogic Server y el resto de la pila de Fusion Middleware.

Para obtener más información, consulte la sección Descripción del modelo de administración de OHS.

Supervisión
Integración con Oracle Enterprise Manager permite a los clientes supervisar el tráfico HTTP utilizando la consola de Oracle Enterprise Management.

Potente Herramienta de Migración
Las herramientas de migración integradas facilitan la migración de los despliegues de Oracle HTTP Server 11g a Oracle HTTP Server 12c.
