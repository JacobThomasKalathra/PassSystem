Oracle HTTP Server 12c  




Oracle HTTP Server 12c 建立在成熟的开源 Apache HTTP Server 技术基础之上，提供了用于托管静态和动态 Web 页以及作为 Oracle Fusion Middleware 应用程序前端的框架。






功能


内容提供/反向代理

云部署/虚拟服务器支持
从单个 Web 服务器实例服务于数千个站点/应用程序域。每个虚拟服务器可以具有各自的配置文件、IP 地址、端口、文档根、首选项、日志文件等等。

针对常见威胁提供保护
内置 ModSecurity 模块提供了配置规则的能力，从而对应用程序进行自测并保护应用程序免遭常见攻击，包括 SQL/命令注入、跨站点脚本攻击以及其他漏洞。

FastCGI 支持
使用脚本语言 (例如 PHP 或 Python) 在 OHS 内部提供动态内容网页的有效方式，而不会造成明显的性能损失。

集成式反向代理
内置代理模块提供了一般代理支持以及针对 WebLogic Server 优化的支持，使 OHS 能够充当 HTTP 原始服务器 (包括 WebLogic Server) 的 HTTP 端点。

管理/监视

服务器管理
充分利用 WebLogic 12c 管理界面提供一个简单、一致的分布式管理模型，以便管理 Oracle HTTP Server、Oracle WebLogic Server 以及 Fusion Middleware 系列的其他组件。

有关详细信息，请参阅了解 OHS 管理模型一节。

监视
与 Oracle Enterprise Manager 相集成，使客户能够使用 Oracle Enterprise Management 控制台监视 HTTP 通信。

强大的移植工具
利用集成式移植工具可以轻松地将现有 Oracle HTTP Server 11g 部署移植到 Oracle HTTP Server 12c。
