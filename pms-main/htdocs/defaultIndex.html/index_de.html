Oracle HTTP Server 12c  




Oracle HTTP Server 12c basiert auf der bewährten Open Source-Apache HTTP Server-Technologie und liefert das Framework für das Hosting statischer, dynamischer Webseiten und das Frontending von Oracle Fusion Middleware-Anwendungen.






Features


Content-Serving/Reverse Proxy

Cloud-Deployment/Unterstützung virtueller Server
Tausende Sites/Anwendungsdomains werden von einer einzelnen Webserverinstanz bedient. Jeder virtuelle Server kann seine eigenen Konfigurationsdateien, IP-Adressen, Ports, Dokument-Roots, Voreinstellungen, Logdateien und mehr haben.

Schutz vor gängigen Bedrohungen
Mit dem integrierten ModSecurity-Modul können Sie Regeln konfigurieren, um Anwendungen zu introspektieren und vor gängigen Angriffen zu schützen, einschließlich SQL-/Befehls-Injection, Cross-Site Scripting-Sicherheitslücken und andere Schwachstellen.

FastCGI-Unterstützung
Effizientes Betreiben von Webseiten mit dynamischem Inhalt in OHS mit Skriptsprachen wie PHP oder Python, ohne erhebliche Beeinträchtigung der Performance.

Integrierter Reverse Proxy
Integrierte Proxymodule bieten generische Proxyunterstützung sowie optimierte Unterstützung für WebLogic Server, sodass OHS als HTTP-Endpunkt für HTTP-Ursprungsserver, einschließlich WebLogic Server, fungieren kann.

Administration/Überwachung

Serveradministration
Nutzt die WebLogic 12c-Administrationsoberflächen, um ein einfaches, konsistentes und verteiltes Administrationsmodell für die Verwaltung von Oracle HTTP Server, Oracle WebLogic Server und dem Rest des Fusion Middleware-Stacks bereitzustellen.

Weitere Informationen finden Sie im Abschnitt OHS-Administrationsmodell.

Überwachung
Dank der Integration in Oracle Enterprise Manager können Kunden den HTTP-Datenverkehr mit der Oracle Enterprise Management-Konsole überwachen.

Robustes Migrationstool
Integrierte Migrationstools erleichtern die Migration vorhandener Oracle HTTP Server 11g-Deployments in Oracle HTTP Server 12c.
